# Практична робота: базові команди Linux

## Завдання 1. Базові команди

| # | Дія | Команда |
|---|------|---------|
| 1 | Вміст домашнього каталогу | `ls ~` |
| 2 | Перейти в Downloads і показати вміст | `cd ~/Downloads && ls` |
| 3 | Створити порожній файл **без `touch`** | `> file.txt` |
| 4 | Переглянути вміст файлу | `cat file.txt` |
| 5 | У домашній каталог **абсолютним** шляхом | `cd /home/name` |
| 6 | У домашній каталог **відносним** шляхом | `cd ..` (якщо ти в `~/Downloads`) |

**Пояснення:**

- **Порожній файл без `touch`** — оператор перенаправлення `>` створює файл нульового
  розміру. Альтернативи: `: > file.txt` або `echo -n > file.txt`.
- **Абсолютний шлях** починається з `/` (від кореня): `/home/name`. Працює з будь-якого місця.
- **Відносний шлях** рахується від поточного каталогу. Якщо ти в `~/Downloads`, то `cd ..`
  піднімає на рівень вище — у домашній каталог. Тут `..` означає «батьківський каталог».

## Завдання 2. Робота з документацією

Команди для виконання:

```bash
man ls
help cd
man cat
man man
cp --help
mv --help
```

### Питання та відповіді

**1. Який ключ `ls` показує приховані файли?**

`ls -a` (від *all*) — показує файли, що починаються з крапки (`.bashrc`, `.config` тощо),
а також `.` і `..`. Близький варіант — `ls -A` (те саме, але без `.` і `..`).

**2. Який ключ у `cat` нумерує рядки?**

`cat -n` — нумерує **усі** рядки. Споріднений `cat -b` — нумерує лише **непорожні** рядки.

**3. Чим відрізняються `man` і `--help`?**

| `man` | `--help` |
|-------|----------|
| Окрема системна сторінка-довідник | Вбудований у саму програму флаг |
| Повний опис: секції, приклади, формати, файли | Коротка довідка зі списком опцій |
| Відкривається у pager (вихід — клавіша `q`) | Просто друкує текст у термінал |
| `man ls` | `ls --help` |

> `help cd` — це довідка для **вбудованих команд оболонки** (bash builtins, як `cd`, `pwd`,
> `echo`). Для них немає окремого `man`, бо вони є частиною самого bash.

## Завдання 3. Міні-сценарій

```bash
cd /tmp                # 1. перейти в каталог
> report.txt           # 2. створити там один файл (без touch)
ls -l                  # 3. переглянути список файлів
cd ~/Downloads         # 4. перейти в інше місце
ls                     #    (перевірити, що ми справді переїхали)
cd -                   # 5. повернутись назад (у /tmp — попередній каталог)
man ls                 # 6. переглянути документацію команди (вихід — q)
```

**Як це працює:**

- `cd -` повертає в **попередній** каталог, де ти був до цього (bash зберігає його у `$OLDPWD`).
- Для пункту 6 можна взяти будь-яку команду зі сценарію: `ls --help`, `help cd`, `man cat`.
- З `man` виходиш клавішею `q`.
