# Практична робота: файлова система, права доступу та користувачі

## Завдання 1. Ієрархія каталогів Linux

| # | Дія | Команда |
|---|------|---------|
| 1 | Перейти в кореневий каталог `/` і показати вміст | `cd / && ls` |
| 2 | Перейти в `/etc` і показати вміст | `cd /etc && ls` |
| 3 | Перейти в `/home` і показати список користувачів | `cd /home && ls` |

**Пояснення:**

- **`/`** — корінь усієї файлової системи. Усі інші каталоги «ростуть» із нього.
  Тут лежать `bin`, `etc`, `home`, `usr`, `var`, `tmp` тощо.
- **`/etc`** — каталог системних **конфігураційних файлів** (налаштування служб,
  користувачів, мережі). Наприклад: `passwd`, `hostname`, `fstab`.
- **`/home`** — містить **домашні каталоги користувачів**. Кожна підпапка тут — це
  окремий користувач, тому `ls /home` фактично показує список людей у системі.

## Завдання 2. Файли, каталоги та посилання

| # | Дія | Команда |
|---|------|---------|
| 1 | Створити новий каталог у домашньому | `mkdir ~/lab2` |
| 2 | Створити всередині файл | `cd ~/lab2 && > file.txt` |
| 3 | Скопіювати файл у нове ім’я | `cp file.txt file_copy.txt` |
| 4 | Перейменувати копію | `mv file_copy.txt renamed.txt` |
| 5 | Створити жорстке посилання | `ln file.txt file_hard.txt` |
| 6 | Створити символічне посилання | `ln -s file.txt file_sym.txt` |
| 7 | Знайти файл по імені | `find ~ -name file.txt` |

**Приклад результату пошуку:**

```
/home/student/lab2/file.txt
```

**Пояснення:**

- **Жорстке посилання (`ln`)** — це другий «вхід» до того самого вмісту на диску
  (того самого inode). Файли рівноправні; видалення одного **не** знищує дані, доки
  існує хоча б одне посилання. Працює лише в межах однієї файлової системи.
- **Символічне посилання (`ln -s`)** — окремий маленький файл-вказівник, що містить
  **шлях** до оригіналу (як ярлик). Якщо видалити оригінал — посилання «зламається»
  (стане «битим»). Може вказувати на інші диски та на каталоги.
- **`find ~ -name file.txt`** шукає за іменем, починаючи з домашнього каталогу.
  Для пошуку по всій системі: `find / -name file.txt 2>/dev/null`
  (`2>/dev/null` приховує помилки доступу до чужих каталогів).

## Завдання 3. Права доступу

| # | Дія | Команда |
|---|------|---------|
| 1 | Переглянути права файлу | `ls -l file.txt` |
| 2 | Надати права тільки на читання | `chmod 444 file.txt` |
| 3 | Надати власнику право на запис | `chmod u+w file.txt` |
| 4 | Переглянути значення `umask` | `umask` |
| 5 | Встановити нове значення | `umask 022` |

**Пояснення:**

- Вивід `ls -l` має вигляд `-rw-r--r-- 1 user user 0 ... file.txt`. Перші 10 символів —
  це тип і права: **власник** (`rw-`), **група** (`r--`), **інші** (`r--`).
- **`chmod 444`** — у вісімковій формі `4=r`, `2=w`, `1=x`. `444` означає `r--r--r--`,
  тобто читання для всіх і нікому не можна писати.
- **`chmod u+w`** — символьна форма: `u` (user/власник) `+w` (додати запис).
  Після цього файл матиме `rw-r--r--`.
- **`umask`** задає, які права **прибираються** у нових файлів за замовчуванням.
  При `umask 022` нові файли отримують `666-022=644` (`rw-r--r--`),
  а нові каталоги — `777-022=755` (`rwxr-xr-x`).

## Завдання 4. Користувачі

| # | Дія | Команда |
|---|------|---------|
| 1 | Створити нового користувача | `sudo useradd -m -s /bin/bash trainee` |
| 2 | Додати його до sudo-групи | `sudo usermod -aG sudo trainee` |
| 3 | Перевірити, що користувач існує | `grep trainee /etc/passwd` |

**Приклад результату:**

```
trainee:x:1001:1001::/home/trainee:/bin/bash
```

**Пояснення:**

- **`useradd -m -s /bin/bash`** — `-m` створює домашній каталог `/home/trainee`,
  `-s /bin/bash` задає оболонку за замовчуванням. Бажано також одразу задати пароль:
  `sudo passwd trainee`.
- **`usermod -aG sudo`** — `-a` (append) **додає** користувача до групи, не стираючи
  попередні; `-G sudo` — цільова група. У Debian/Ubuntu група адміністраторів зветься
  `sudo`, а в RHEL/CentOS/Fedora — `wheel`.
- **Перевірка існування** — рядок у `/etc/passwd` розшифровується так:
  `ім'я:пароль(x у /etc/shadow):UID:GID::домашній_каталог:оболонка`.
  Альтернатива: `id trainee` покаже `uid`, `gid` та групи (зокрема `sudo`).
